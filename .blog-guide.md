# Blog Post Creation Guide
*Private guide for creating Jekyll blog posts on harshilkamdar.github.io*

## Overview
Your blog uses Jekyll's built-in post system. Posts are written in Markdown and stored in the `_posts/` directory. The blog is accessible at `/notes/` and uses the `post-list.html` include for listing posts.

---

## Step-by-Step Blog Post Creation

### Step 1: Create the Post File
**File Naming Convention:** `YYYY-MM-DD-post-title.markdown` or `.md`

```bash
# Navigate to _posts directory
cd _posts/

# Create new post file (example)
touch 2025-12-10-my-new-post.markdown
```

**Important:** 
- Date format must be `YYYY-MM-DD`
- Use hyphens (not spaces) in filename
- Extension can be `.md`, `.markdown`, or `.html`

### Step 2: Add Front Matter
Every post must start with YAML front matter:

```yaml
---
layout: post
title: "Your Post Title Here"
date: 2025-12-10
published: true
tags: [tag1, tag2, optional]
---
```

**Front Matter Options:**
- `layout: post` - **Required** (uses `_layouts/post.html`)
- `title: "..."` - **Required** (post title)
- `date: YYYY-MM-DD` - **Required** (publication date)
- `published: true/false` - **Optional** (default: true)
- `tags: [...]` - **Optional** (post tags)

### Step 3: Write Content
After front matter, write your content in Markdown:

```markdown
---
layout: post
title: "My Awesome Blog Post"
date: 2025-12-10
published: true
tags: [tech, jekyll]
---

## Introduction

This is my blog post content written in **Markdown**.

### Subsection

- Bullet points work
- As expected

You can add [links](https://example.com) and `inline code`.

```code blocks
work too
```

Images can be added:
![Alt text](/files/image.png)
```

### Step 4: Preview Locally
Test your post before publishing:

```bash
# Build and serve locally
bundle exec jekyll serve

# Visit http://127.0.0.1:4000/notes/ to see blog
# Visit http://127.0.0.1:4000/YYYY/MM/DD/post-title/ for individual post
```

### Step 5: Add Media Files (if needed)
Store images/files in the `/files/` directory:

```bash
# Copy files to public directory
cp my-image.png files/
```

Reference in post:
```markdown
![Description](/files/my-image.png)
[Download File](/files/document.pdf)
```

### Step 6: Commit and Publish
Once satisfied with the post:

```bash
# Add files
git add _posts/2025-12-10-my-new-post.markdown
git add files/any-new-files.png  # if applicable

# Commit
git commit -m "Add new blog post: Title of Post"

# Push to publish
git push origin master
```

## Advanced Features

### Tags
Add tags to categorize posts:
```yaml
tags: [research, astronomy, data]
```

### Draft Posts
Create unpublished drafts:
```yaml
published: false
```

### Custom Excerpts
Add post excerpts for better previews:
```yaml
excerpt: "This is a brief description of the post..."
```

### Date Control
Override filename date:
```yaml
date: 2025-12-10 14:30:00 -0800
```

## Troubleshooting

### Common Issues:
1. **Post not appearing:** Check filename date format
2. **Build errors:** Validate YAML front matter syntax
3. **Missing styling:** Ensure `layout: post` is set
4. **Links broken:** Use relative paths for internal content

### Testing Checklist:
- [ ] Filename follows `YYYY-MM-DD-title.ext` format
- [ ] YAML front matter is valid
- [ ] Content renders properly locally
- [ ] Links and images work
- [ ] Post appears in `/notes/` listing
